About

A 2D tower defense survival game in which the player must manage resources and prepare for upcoming challenges. Players will need to make critical decisions, such as searching supplies, crafting and engaging in trade, to ensure their survival.

Project Info

Role Game Designer and Team manager
Team Size 3 Members
Engine Unity
Development Time WIP

What I Do

  • I am a designer, team manager, and developer.
  • I design mechanics and plan map layouts for expedition maps.
  • I create NPC class diagrams to simplify development for the team.
  • I define item stats such as rarity, trade value, disassembly value, and quality.
  • I design how Dynamic Difficulty Adjustment (DDA) responds to player actions, item interactions, progression, and player kills.
  • I design the UI layout, detailing what elements should be included and how they should interact with the player.
  • Developed the Main Spawner mechanics:Decided what to spawn and how many.and Determined which lane to spawn in and the delay between each spawn.
  • Developed the Crafting System:Designed what items can be crafted and how long crafting takes and Defined the required recipes and ensured players have the necessary materials to craft items.

What I learned from this project

This project is still in progress, so I haven't learned much from it yet, but I did gain an important insight about time mechanics in games. Time mechanics need to be developed carefully, especially if they are integral to multiple aspects of the game. For example, functions like fast-forward, time scaling, and timestamp checks need to work in sync with other systems. A timestamp function should return true when a specific time is reached, allowing other functions to proceed smoothly.
